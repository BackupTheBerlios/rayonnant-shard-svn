Rayonnant Shard Release Information

General Information
The Rayonnant Shard will have three types of releases:
  • Map file releases.
  • Definitions and scripts releases.
  • Package or phase based releases.
Map File Releases
The map files are a vital part to this project. We will have timestamped releases with these files. Releases will be provided with BitTorrents to promote a shared release system due to these being the largest of the file releases.

Example archived map release: shard-map_YYYYMMDD.tar.bz2
Example torrent map release: shard-map_YYYYMMDD.tar.bz2.torrent

BitTorrents can be made with the following command:
btmakemetafile.py shard-map_YYYYMMDD.tar.bz2 \
    http://rayonnant.net:7000/announce \
    --comment "http://shard.rayonnant.net"
Definition and Script Releases
The definitions and scripts are another vital part of the project. These are also developed sporadicly so we will also be using a timestamped release method here.

Example archived scripts release: shard-scripts_YYYYMMDD.tar.bz2
Phase Based Releases
The Rayonnant Shard has three phases layed out:
  • Phase 1: Building of the world's map data.
  • Phase 2: Scripting the world's events and objects.
  • Phase 3: Bug hunting and general testing.
Due to the development style of this project, we will have packaged releases based on a phase and timestamp method. It is possible for releases to be between phases. However we will not be between phase 1 and phase3.

An example of a phase release: shard_phase1_20050306.tar.bz2
An example of a mix phased release: shard_phase1-2_20050306.tar.bz2
